Primitive completion, cannot be used with most transformers (like ic_complete_word and ic_complete_qword).

When completed, delete_before bytes are deleted before the cursor position, delete_after bytes are deleted after the cursor, and finally completion is inserted. The display is used to display the completion in the completion menu, and help is displayed with hinting. Both display and help can be NULL. (all are copied by isocline and do not need to be preserved or allocated).

Returns true if the callback should continue trying to find more possible completions. If false is returned, the callback should try to return and not add more completions (for improved latency).
